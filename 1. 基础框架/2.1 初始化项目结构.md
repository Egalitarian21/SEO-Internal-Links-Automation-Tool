# Phase 1-1：初始化项目结构

## 任务描述
用一条结构化 Prompt 让 AI 生成完整的项目骨架，包含前台页面路由、
后台 Dashboard 路由、API 路由结构、数据库连接配置和环境变量模板。
这一步决定了整个项目的目录结构，后续所有开发都在此基础上展开。

---

## Prompt

请帮我初始化一个 Next.js 15 + TypeScript + Tailwind v4 + shadcn/ui 
的全栈项目，包含以下完整结构：

目录结构要求：
- /app
  - (public)/          # 公开页面（首页、登录、注册）
  - (dashboard)/       # 需要登录的后台页面
    - dashboard/       # 数据概览
    - projects/        # 项目管理
    - settings/        # 用户设置
  - api/               # API 路由
    - auth/            # 认证相关
    - projects/        # 项目 CRUD
    - users/           # 用户管理
- /components
  - ui/                # 基础 UI 组件（shadcn）
  - layout/            # 布局组件（Sidebar, Header, Footer）
  - features/          # 业务功能组件
- /lib
  - db.ts              # Prisma 客户端
  - auth.ts            # 认证工具
  - utils.ts           # 通用工具函数
  - validations/       # Zod 验证 schema
- /prisma
  - schema.prisma      # 数据库模型
  - seed.ts            # 种子数据
- .env.example         # 环境变量模板
- README.md            # 项目说明

请同时生成：
1. package.json（含所有必要依赖）
2. tsconfig.json（严格模式）
3. tailwind.config.ts
4. .env.example（列出所有需要的环境变量，值留空）
5. README.md（含本地启动步骤）
