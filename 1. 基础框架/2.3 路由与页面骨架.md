# Phase 1-3：路由与页面骨架

## 任务描述
让 AI 一次性生成所有页面的空壳文件，只包含布局结构和导航逻辑，
不包含任何业务逻辑。建立完整的导航树，确保页面间跳转关系正确，
为后续逐页填充内容打好基础。

---

## Prompt

请为项目生成所有页面的骨架文件，要求：

页面清单：
- / (首页 - 产品介绍 + CTA)
- /login (登录页)
- /register (注册页)
- /dashboard (数据概览)
- /projects (项目列表)
- /projects/[id] (项目详情)
- /projects/new (新建项目)
- /settings (用户设置)
- /404 (自定义错误页)

每个页面骨架包含：
1. 正确的 Next.js 15 文件结构（page.tsx + layout.tsx）
2. 页面标题（metadata export）
3. 基础布局占位（用注释标明各区域用途）
4. 页面间的 Link 导航关系
5. 加载状态文件（loading.tsx）
6. 错误边界文件（error.tsx）

所有数据区域使用注释占位，例如：
{/* TODO: 项目列表数据将在 Phase 4 接入 */}

同时生成 /components/layout/Navigation.tsx，
包含所有页面的导航菜单配置（路径 + 图标 + 名称）。
