# Phase 2-1：页面级布局生成

## 任务描述
逐页生成完整的 UI 布局，每次 Prompt 聚焦一个页面，使用 Mock 数据填充。
遵循"从粗到细，从静到动"原则：先完成视觉布局，再添加交互逻辑。
使用 shadcn/ui 组件库保证设计一致性。

---

## Prompt

生成 /dashboard 页面的完整 UI 布局：

页面结构：
- 左侧固定导航栏（宽度 240px）
  - 顶部：Logo + 产品名称
  - 中部：5个菜单项（图标 + 文字 + 激活状态）
  - 底部：用户头像 + 名称 + 退出按钮
- 顶部 Header（高度 64px）
  - 左侧：当前页面面包屑
  - 右侧：搜索框 + 通知铃铛（带红点） + 用户头像下拉菜单
- 主内容区
  - 第一行：4个数据统计卡片（总项目数 / 本月新增 / 活跃用户 / 完成率）
  - 第二行：折线图（左，占 2/3 宽）+ 环形图（右，占 1/3 宽）
  - 第三行：数据表格（项目列表，含分页）

技术要求：
- 使用 shadcn/ui 的 Card / Table / Badge 等组件
- 所有数据使用 Mock 数据（在文件顶部定义 const mockData = {...}）
- 使用 Tailwind CSS 实现布局，不使用自定义 CSS
- 添加 TypeScript 类型定义

请同时说明：哪些部分建议提取为独立组件？
