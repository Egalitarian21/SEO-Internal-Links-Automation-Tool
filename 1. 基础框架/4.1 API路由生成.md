# Phase 3-1：API 路由生成（逐模块）

## 任务描述
按业务模块逐一生成后端 API 路由，每个模块一个 Prompt，避免上下文过长导致质量下降。
每个 API 路由需要包含完整的 CRUD 操作、输入验证、错误处理和统一响应格式。
使用 Zod 进行请求体验证，确保类型安全。

---

## Prompt

生成 /api/projects 模块的完整 CRUD API：

接口清单：
- GET    /api/projects          # 获取项目列表（分页 + 搜索 + 过滤）
- GET    /api/projects/[id]     # 获取单个项目详情
- POST   /api/projects          # 创建新项目
- PUT    /api/projects/[id]     # 更新项目信息
- DELETE /api/projects/[id]     # 删除项目

每个接口要求：
1. 使用 Zod 定义请求参数/请求体的验证 Schema（放在 /lib/validations/project.ts）
2. 使用 Prisma 操作数据库
3. 统一响应格式：
   成功：{ success: true, data: T, message: string }
   失败：{ success: false, error: string, code: number }
4. 错误处理覆盖：
   - 400 参数验证失败（附带字段级错误详情）
   - 401 未登录
   - 403 无权限
   - 404 资源不存在
   - 500 服务器内部错误

GET /api/projects 特殊要求：
- 支持 ?page=1&limit=10 分页参数
- 支持 ?search=关键词 模糊搜索（搜索 title 和 description）
- 支持 ?status=active|archived 状态过滤
- 响应包含 pagination: { total, page, limit, totalPages }
