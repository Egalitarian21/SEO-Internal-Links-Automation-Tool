# Phase 3-3：数据库迁移与种子数据

## 任务描述
生成数据库迁移命令序列和种子数据脚本，用于初始化开发和测试环境。
种子数据应覆盖所有主要实体，并建立合理的关联关系，
方便开发阶段进行 UI 联调和功能测试。

---

## Prompt

请生成数据库初始化的完整方案：

第一部分：迁移命令序列
生成一个 scripts/db-setup.sh 脚本，包含：
1. 创建数据库
2. 运行 prisma migrate dev
3. 运行种子数据脚本
4. 验证数据是否正确插入

第二部分：种子数据脚本（prisma/seed.ts）
生成以下测试数据：

用户数据（3个）：
- admin@test.com（角色：admin）
- editor@test.com（角色：editor）  
- viewer@test.com（角色：viewer）
- 密码统一：Test123456!

项目数据（10条）：
- 覆盖所有状态（active / archived / draft）
- 不同的创建时间（跨越最近3个月）
- 关联不同的创建者

关联数据：
- 每个项目 2-4 个标签（Tag）
- 每个项目 3-5 条评论（Comment）
- 评论来自不同用户

要求：
- 使用 faker.js 生成真实感的随机数据
- 种子脚本支持重复执行（先清空再插入）
- 添加执行进度日志输出
- 生成完成后打印数据统计摘要
