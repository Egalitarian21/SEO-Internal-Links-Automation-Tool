# Phase 4-2：替换 Mock 数据

## 任务描述
将前端页面中的静态 Mock 数据逐一替换为真实 API 调用。
这是前后端联调的核心步骤，需要同时处理加载态、空状态和错误状态的 UI 展示。
每次只替换一个页面，确保联调质量。

---

## Prompt

将 Dashboard 页面的 Mock 数据替换为真实 API 调用：

替换清单：

1. 统计卡片数据
   - 调用 GET /api/stats 获取 { totalProjects, monthlyNew, activeUsers, completionRate }
   - 加载时显示 Skeleton（4个卡片骨架）
   - 请求失败时显示错误状态 + 重试按钮

2. 项目列表表格
   - 调用 GET /api/projects?limit=10&page=1
   - 加载时显示表格 Skeleton（10行骨架）
   - 数据为空时显示 EmptyState 组件（含"创建第一个项目"按钮）
   - 支持表格内分页切换

3. 折线图数据
   - 调用 GET /api/stats/trend?period=30d 获取趋势数据
   - 加载时图表区域显示 Skeleton
   - 数据加载完成后使用动画渲染图表

状态处理要求：
- 所有 loading 状态使用 Skeleton 而非 Spinner（更好的用户体验）
- 错误状态显示具体错误原因（开发环境）或通用提示（生产环境）
- 使用 React Suspense + ErrorBoundary 包裹数据区域
- 添加 SWR revalidate on focus（切换 Tab 后自动刷新）
