# Phase 5-1：认证集成

## 任务描述
集成 Clerk 认证系统，实现完整的用户登录、注册流程。
保护所有需要登录的路由，在 API 层验证用户身份。
处理用户首次登录时的数据库记录创建（Webhook 同步）。

---

## Prompt

集成 Clerk 认证系统，实现完整的认证流程：

前端认证页面：
1. /login 页面
   - 使用 Clerk 的 <SignIn /> 组件
   - 自定义样式匹配项目设计（appearance prop）
   - 登录成功后跳转 /dashboard
   - 添加"忘记密码"链接

2. /register 页面
   - 使用 Clerk 的 <SignUp /> 组件
   - 注册成功后跳转 /dashboard（首次引导流程）

路由保护：
- 在 middleware.ts 中配置 Clerk 路由保护
- 保护所有 /dashboard/* 路由
- 保护所有 /api/* 路由（除 /api/public/* 外）
- 未登录访问受保护路由 → 重定向到 /login

用户数据同步（Webhook）：
- 创建 /api/webhooks/clerk/route.ts
- 监听 user.created 事件 → 在数据库创建 User 记录
- 监听 user.updated 事件 → 同步更新数据库
- 监听 user.deleted 事件 → 软删除数据库记录
- 验证 Webhook 签名（使用 svix）

用户 Profile：
- 创建 /lib/auth.ts 工具函数：getCurrentUser() 获取当前登录用户的数据库记录
- 在 Layout 中获取用户信息并传递给 Header 组件
