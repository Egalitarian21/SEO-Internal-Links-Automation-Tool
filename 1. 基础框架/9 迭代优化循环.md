# Phase 8：迭代优化循环

## 任务描述
产品上线后进入持续迭代模式。这是 Vibe Coding 真正发挥威力的阶段：
用自然语言描述用户反馈和新需求，让 AI 快速定位代码并实现修改。
人工负责需求判断、方案选择和代码审查，AI 负责具体实现。

---

## Prompt 模板集合

### 模板 A：Bug 修复
```
我发现了一个 Bug，描述如下：
[粘贴用户反馈或错误截图描述]

错误信息（如有）：
[粘贴 console error 或 server log]

复现步骤：
1. [步骤1]
2. [步骤2]
3. [步骤3]

请：
1. 定位可能出问题的代码位置
2. 分析根本原因
3. 提供修复方案（如有多种，列出利弊）
4. 实现修复代码
5. 说明如何验证修复有效
```

### 模板 B：新功能开发
```
需要添加新功能：[功能描述]

用户故事：作为 [用户角色]，我希望 [做什么]，以便 [达到什么目的]

验收标准：
- [ ] [条件1]
- [ ] [条件2]
- [ ] [条件3]

约束条件：
- 不能影响现有 [相关功能]
- 需要兼容移动端
- 性能要求：[如有]

请先给出实现方案（不要直接写代码），
确认方向后再实现。
```

### 模板 C：代码重构
```
以下代码需要重构：
[粘贴代码]

重构目标：
- [ ] 提高可读性
- [ ] 减少重复代码
- [ ] 提升性能
- [ ] 添加类型安全

约束：
- 保持对外接口不变（不能改变函数签名）
- 不能引入新的依赖
- 重构后需要通过现有测试

请提供重构后的代码，并说明每处改动的理由。
```

### 模板 D：性能问题排查
```
用户反馈 [页面/功能] 很慢，具体表现：
[描述慢的现象]

已知信息：
- 数据量：[大概多少条数据]
- 用户网络：[正常/慢速]
- 发生频率：[总是/偶尔]

请帮我：
1. 列出可能的性能瓶颈（按可能性排序）
2. 提供排查方法（如何确认是哪个原因）
3. 针对最可能的原因提供优化方案
```

---

## 迭代循环流程

```
用户反馈 / 新需求
      ↓
用自然语言描述问题（使用上方模板）
      ↓
AI 定位代码 + 提出 2-3 种方案
      ↓
人工选择方向（30秒决策）
      ↓
AI 实现代码
      ↓
本地验证（跑测试 + 手动测试）
      ↓
Code Review（重点看逻辑，不看格式）
      ↓
合并 + 自动部署
      ↓
下一个迭代
```
