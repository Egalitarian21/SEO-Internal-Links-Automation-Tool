# 博客内链 Prompt 流水线说明

这个目录包含一套用于 SEO 博客内部链接体系搭建的 Prompt 流水线。  
目标是把“内容理解 -> 候选召回 -> 链接生成 -> 质检 -> 发布审计 -> 周度复盘”串成可执行闭环。

## 目录与执行顺序

1. `SEO架构师-定义内链评分函数.md`
2. `内容结构化引擎-文章结构化抽取.md`
3. `链接规划器-候选目标页排序.md`
4. `SEO编辑-生成内链插入建议.md`
5. `内部链接质检器-审核链接建议.md`
6. `SEO图谱审计员-发布前检查.md`
7. `内链系统复盘助手-分析周度数据并输出迭代建议.md`

## 每一步输入与输出

### Step 1: 定义评分函数
- 文件：`SEO架构师-定义内链评分函数.md`
- 输入：站点类型、文章规模、业务目标
- 输出：量化指标、总分公式、上线阈值、人工复核阈值
- 用途：作为后续所有步骤的统一评估标准

### Step 2: 文章结构化抽取
- 文件：`内容结构化引擎-文章结构化抽取.md`
- 输入：`title`、`url`、`content`
- 输出：结构化 JSON（主题、关键词、可链接段落、锚文本候选等）
- 用途：生成“可链接对象”，为召回与重排提供数据基础

### Step 3: 候选目标页排序
- 文件：`链接规划器-候选目标页排序.md`
- 输入：源文章信息 + 候选目标页列表（通常来自向量召回/BM25）
- 输出：Top-K 目标页及相关性分数、推荐锚文本、插入位置建议
- 用途：把“可能可链”收敛为“优先可链”

### Step 4: 生成插入建议
- 文件：`SEO编辑-生成内链插入建议.md`
- 输入：源文章正文 + 链接计划 JSON
- 输出：可落地的插入方案（锚文本、前后片段、最终句子）
- 用途：从规划结果生成可发布草案

### Step 5: 质检审核
- 文件：`内部链接质检器-审核链接建议.md`
- 输入：上下文、锚文本、目标页摘要、生成理由
- 输出：`pass|reject|review`、风险标签、修复建议
- 用途：拦截语义漂移、过度优化、可读性问题

### Step 6: 发布前检查
- 文件：`SEO图谱审计员-发布前检查.md`
- 输入：页面元数据、链接边数据、策略阈值
- 输出：通过/阻塞结论、阻塞项、修复动作、影响预测
- 用途：作为发布闸门，避免批量错误上线

### Step 7: 周度复盘迭代
- 文件：`内链系统复盘助手-分析周度数据并输出迭代建议.md`
- 输入：点击日志、收录/排名变化、质量信号、上周策略参数
- 输出：有效项、失败项、尾部失败样本、参数更新建议、A/B 测试计划
- 用途：形成持续优化闭环

## 推荐数据结构（最小集）

建议至少维护以下 4 类数据：

- `pages`：页面元数据（url、title、topic、intent、quality）
- `page_features`：结构化抽取结果（关键词、可链接段落、锚文本候选）
- `link_plan`：候选与重排结果（source -> target、score、reason）
- `link_graph`：最终上线链接（anchor、status、quality_score、timestamps）

## 串联建议（工程实践）

- 用统一 `job_id` 贯穿单次批处理，便于追踪每一步产物
- 每一步都落盘 JSON（不要只保留最终结果）
- 对 `reject/review` 建立人工复核队列
- 先小流量试点（如 200 篇）再全量推广

## 快速开始

1. 先跑 Step 1，固定评分函数与阈值  
2. 批量跑 Step 2 生成全站结构化数据  
3. 对每篇文章执行 Step 3 -> Step 4 -> Step 5  
4. 汇总后执行 Step 6，只有通过才发布  
5. 每周执行 Step 7，更新规则与参数

## 注意事项

- Prompt 输出应强制 JSON（便于程序消费）
- 生成模型与质检模型建议分离，避免自我确认偏差
- 严控锚文本重复率与关键词堆砌风险
- 重点盯最差 10% 样本，而不是平均表现
