## 一、一句话产品定位

> 一个基于 LLM 语义理解与 Wiki 知识图谱的电商博客内链自动化系统，用于批量发现、校验并管理符合 SEO 规范的高质量内部链接。本地团队使用，输出结果直接链接到Shopify后台

---

## 二、核心用户故事

**Story 1 — 语义发现**
- **As a** SEO 实施工程师，
- **I want** 系统自动理解每篇博客的核心主题与产品意图，
- **So that** 我无需手动阅读全文，即可获得语义相关的候选内链推荐列表。

---

**Story 2 — 规范落地**
- **As a** SEO 实施工程师，
- **I want** 按照预设的内链规范（锚文本多样性、链接密度上限、避免孤岛页面）自动过滤推荐结果，
- **So that** 每一条插入的内链都符合站点 SEO 标准，无需二次人工校验。

---

**Story 3 — 批量提效**
- **As a** SEO 实施工程师，
- **I want** 对整个博客库执行一键批量扫描并生成可审计的内链变更报告，
- **So that** 我可以在 30 分钟内完成原本需要数天的博客集群内链梳理工作。

---

## 三、功能范围

### ✅ 必须有（Must Have）

- **Wiki 式知识图谱视图**
  - 以节点图可视化博客集群的内链拓扑，直观呈现链接密度与孤岛分布
  - 核心底层逻辑为文章节点之间的关系由 LLM 基于博客wiki卡片数据库:
  	1. 读一篇博客
	2. 产出这篇博客的 wiki 卡片
	3. 根据锚文本规范，找到适合做内链的词
	4. 在已有博客库里找最适合链接过去的文章
	5. 不断把整站博客整理成主题内容集群

- **内链候选推荐**
  - 基于语义相似度 + 产品主题匹配，为每篇文章推荐 Top-N 候选内链
  - 推荐结果包含：目标 URL、建议锚文本、插入段落位置、语义相关度评分

- **规范校验层**
  - 锚文本多样性检测（避免重复锚文本堆砌）
  - 单篇文章内链密度上限控制（可配置阈值）
  - 孤岛页面检测与优先补链提示

- **变更报告输出**
  - 生成结构化 CSV / JSON 报告，字段含：文章 ID、插入位置、锚文本、目标链接、置信度
  - 支持「已接受 / 已拒绝 / 待审核」三态标记

- **最小化 Web UI**
  - 任务触发入口（单篇 / 批量）
  - 推荐列表展示与人工一键确认 / 驳回

---

###  应该有（Should Have）

- **增量扫描模式**
  - 新文章发布后自动触发局部重新索引，仅对受影响节点重新推荐，避免全量重跑

- **锚文本生成建议**
  - LLM 根据上下文自动生成 2-3 个自然语言锚文本备选，供工程师选择

- **CMS 集成适配器**
  - 提供 WordPress REST API / Shopify Blog API 的写回接口，支持一键将确认的内链直接更新到 CMS

- **规范配置面板**
  - 可视化配置内链密度、锚文本黑名单、产品集群映射关系等规则参数

---

## 四、核心差异化说明

Link Whisper 和 Internal Link Juicer 的底层逻辑是**关键词匹配 + 规则引擎**，
本工具的核心差异在于引入 **LLM-Wiki 语义推理层**：

- 文章之间的关联不依赖关键词共现，而是由 LLM 理解**内容意图与产品语境**后推断
- 推荐逻辑可解释：每条推荐附带 LLM 生成的「关联理由」，工程师可审计决策依据
- 随博客库扩张，语义索引持续自我优化，推荐质量不因内容规模增长而下降
