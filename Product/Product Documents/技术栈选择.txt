**项目定位：** LLM + Wiki 知识图谱驱动的电商博客内链自动化系统
**约束条件：** 单人开发 · 40天上线 · 免费优先

## 技术栈总览

| 维度       | 选择                        | 
|------------|-----------------------------|
| 前端框架   | Next.js 15                  |
| UI 组件库  | shadcn/ui                   | 
| 后端运行时 | Python + FastAPI            | 
| 数据库     | Supabase (含 pgvector)      | 
| 认证方案   | Auth.js v5                  | 
| 部署平台   | Vercel + Railway            | 

---

## 架构关键说明

- **向量检索是核心：** Supabase pgvector 直接承载语义向量存储与检索，避免引入独立向量数据库，是单人项目控制复杂度的关键决策
- **前后端分离是必要的：** LLM 批量处理任务耗时长，必须运行在支持持久进程的平台（Railway），不能放在 Vercel Serverless 函数中
- **Python 后端不可替代：** Wiki 知识图谱构建、LLM 编排、NLP 文本处理三项核心能力均依赖 Python 生态，强行用 Node.js 实现将消耗大量时间造轮子
