# Phase 0-2：技术栈预选

## 任务描述
在开始 Vibe Coding 前，由人工确定技术边界，避免 AI 在开发过程中随机选型导致技术栈混乱。
需要覆盖前端框架、UI 组件库、后端运行时、数据库、认证方案和部署平台六个维度。
技术选型原则：优先选择 AI 训练数据覆盖广、文档清晰、有托管服务的方案。

---

## Prompt

请根据以下项目背景，为我推荐最合适的技术栈组合，并说明每项选择的理由,输出为.md文档到聊天区：

项目背景：[粘贴你的产品一句话定义]
团队规模：[单人 / 2-3人小团队]
上线周期：[2周 / 1个月 / 3个月]
预算范围：[免费优先 / 小额付费可接受 / 企业级预算]

请按以下维度给出推荐：
- 前端框架（从 Next.js 15 / Nuxt 4 / SvelteKit 中选择）
- UI 组件库（从 shadcn/ui / Radix / Ant Design 中选择）
- 后端运行时（从 Node.js+Hono / Python+FastAPI 中选择）
- 数据库（从 Supabase / PlanetScale / Neon 中选择）
- 认证方案（从 Clerk / Auth.js v5 中选择）
- 部署平台（从 Vercel / Railway / Fly.io 中选择）

输出格式：bulletin形式，包含「选择」「备选」「选择理由」。
