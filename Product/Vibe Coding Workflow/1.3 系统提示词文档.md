# Phase 0-3：系统提示词文档（CLAUDE.md / .cursorrules）

## 任务描述
创建项目级别的 AI 行为约束文件，这是 Vibe Coding 的"宪法文件"。
该文件将被放置在项目根目录，让 AI 在整个开发过程中保持一致的代码风格、
技术选型和行为边界。这是防止 AI 在长期开发中"跑偏"的核心机制。

---

## Prompt

请帮我生成一份完整的 CLAUDE.md 项目配置文件，内容包含：

项目信息：
- 项目名称：[项目名]
- 项目描述：[一句话描述]
- 技术栈：[填入 Phase0-2 确定的技术栈]

请生成以下章节：
1. 项目背景与目标（2-3句话）
2. 技术栈约定（列出所有已选技术，禁止引入未列出的依赖）
3. 代码风格规范
   - 文件命名规则（组件用 PascalCase，工具函数用 camelCase）
   - 注释语言（中文 / 英文）
   - TypeScript 严格模式要求
   - 导入顺序规范
4. 目录结构说明（列出 /app /components /lib /api 各目录用途）
5. 禁止行为清单
   - 不自动安装未经确认的 npm 包
   - 不修改 .env 文件中已有的变量名
   - 不删除已有的注释
   - 不使用 any 类型
6. 响应格式要求（每次修改后说明改了哪些文件）
