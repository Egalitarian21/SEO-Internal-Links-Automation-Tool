# Phase 1-2：数据模型设计（Schema First）

## 任务描述
在编写任何业务代码之前，先让 AI 生成数据库 ER 图描述，经人工审查确认后
再生成实际的 Prisma Schema 代码。这是防止后期大规模返工的关键步骤。
遵循"Schema First"原则：数据模型是整个应用的地基，必须稳固。

---

## Prompt

### 第一步：生成 ER 图描述（人工审查用）

根据以下业务需求，设计数据库实体关系模型：

业务需求：[粘贴你的产品功能描述]

请先以文字形式描述 ER 图，包含：
1. 所有实体（Entity）及其核心字段
2. 实体间的关系（一对一 / 一对多 / 多对多）
3. 每个关键字段的数据类型和约束
4. 需要建立索引的字段及原因

暂不生成代码，等我确认 ER 图后再继续。

---

### 第二步：生成 Prisma Schema（确认后执行）

ER 图已确认，请生成完整的 prisma/schema.prisma，要求：
- 使用 PostgreSQL provider
- 所有模型包含 createdAt / updatedAt 时间戳
- 添加适当的数据库索引（@@index）
- 多对多关系使用显式中间表
- 每个模型添加中文注释说明用途
- 枚举类型单独定义
