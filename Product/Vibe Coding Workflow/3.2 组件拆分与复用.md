# Phase 2-2：组件拆分与复用

## 任务描述
在 AI 生成页面代码后，人工识别可复用的 UI 模式，再让 AI 进行组件提取和重构。
这一步是保证代码可维护性的关键，避免相同 UI 逻辑在多个页面重复出现。
提取的组件需要有完整的 TypeScript Props 类型定义。

---

## Prompt

请将 Dashboard 页面代码中的可复用部分提取为独立组件：

需要提取的组件：

1. StatCard 组件 → /components/features/StatCard.tsx
   Props: title(string), value(string|number), trend(number), 
          trendLabel(string), icon(React.ReactNode), color(string)
   
2. DataTable 组件 → /components/ui/DataTable.tsx  
   Props: columns(ColumnDef[]), data(T[]), pagination(boolean),
          onRowClick((row: T) => void)
   
3. PageHeader 组件 → /components/layout/PageHeader.tsx
   Props: title(string), description(string), 
          actions(React.ReactNode), breadcrumbs(BreadcrumbItem[])

4. EmptyState 组件 → /components/ui/EmptyState.tsx
   Props: icon(React.ReactNode), title(string), 
          description(string), action(React.ReactNode)

要求：
- 每个组件文件包含：类型定义 + 组件实现 + 默认 Props
- 添加 JSDoc 注释说明组件用途和 Props 含义
- 组件内部不包含任何业务逻辑，只负责展示
- 提取完成后，重写 Dashboard 页面使用这些组件
