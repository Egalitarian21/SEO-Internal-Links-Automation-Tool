# Phase 3-2：中间件与鉴权

## 任务描述
为所有 API 路由添加统一的中间件层，包括身份验证、请求频率限制和统一错误处理。
中间件应该是可组合的高阶函数，方便在不同路由中按需使用。
使用 Clerk 的 auth() helper 进行 JWT 验证。

---

## Prompt

为项目 API 路由添加完整的中间件系统：

需要实现的中间件（放在 /lib/middleware/ 目录）：

1. withAuth 中间件（/lib/middleware/withAuth.ts）
   - 使用 Clerk 的 auth() 验证 JWT Token
   - 验证失败返回 401 + 错误信息
   - 验证成功将 userId 注入 request context
   - 支持可选认证模式（某些接口允许未登录访问）

2. withRateLimit 中间件（/lib/middleware/withRateLimit.ts）
   - 基于 IP 地址限流
   - 默认：100次/分钟（可配置）
   - 超限返回 429 + Retry-After header
   - 使用内存存储（开发环境）/ Redis（生产环境）

3. withErrorHandler 中间件（/lib/middleware/withErrorHandler.ts）
   - 捕获所有未处理的异常
   - Zod 验证错误 → 400 + 字段级错误详情
   - Prisma 错误 → 对应 HTTP 状态码
   - 其他错误 → 500 + 隐藏内部细节

4. 中间件组合工具（/lib/middleware/compose.ts）
   - 支持链式组合：withAuth(withRateLimit(handler))
   - 或使用数组形式：compose([withAuth, withRateLimit])(handler)

请同时更新 /api/projects 路由，应用上述中间件。
