# Phase 4-1：数据层封装（API Client）

## 任务描述
创建统一的前端 API 调用层，封装所有与后端的通信逻辑。
使用 TanStack Query 管理服务端状态，统一处理 loading、error、success 状态。
添加请求拦截器自动附加认证 Token，响应拦截器统一处理错误。

---

## Prompt

创建完整的前端 API Client 层：

第一步：基础 HTTP 客户端（/lib/api/client.ts）
- 基于 fetch API 封装，支持 TypeScript 泛型
- 自动附加 Content-Type: application/json
- 使用 Clerk 的 getToken() 自动附加 Bearer Token
- 统一处理响应：解析 JSON，抛出业务错误
- 支持请求超时（默认 10s）
- 开发环境打印请求日志

第二步：业务 API 函数（/lib/api/projects.ts）
封装所有 projects 相关 API 调用：
- getProjects(params: GetProjectsParams): Promise<PaginatedResponse<Project>>
- getProject(id: string): Promise<Project>
- createProject(data: CreateProjectInput): Promise<Project>
- updateProject(id: string, data: UpdateProjectInput): Promise<Project>
- deleteProject(id: string): Promise<void>

第三步：TanStack Query Hooks（/hooks/useProjects.ts）
- useProjects(params)：列表查询，支持分页和过滤
- useProject(id)：单个查询
- useCreateProject()：创建 mutation，成功后自动刷新列表
- useUpdateProject()：更新 mutation
- useDeleteProject()：删除 mutation，含乐观更新

每个 Hook 需要：
- 正确的 queryKey 设计（支持缓存失效）
- loading / error / data 状态
- 错误时显示 toast 通知（使用 sonner）
