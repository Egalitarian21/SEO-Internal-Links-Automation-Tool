# Phase 4-3：表单与写操作

## 任务描述
为所有数据写入操作（创建、编辑、删除）实现完整的表单逻辑。
包括前端验证、提交状态管理、成功/失败反馈和数据刷新。
使用 react-hook-form + zod 实现类型安全的表单验证。

---

## Prompt

为"新建项目"功能实现完整的表单逻辑：

表单字段：
- title（必填，2-50字符）
- description（选填，最多500字符，富文本编辑器）
- status（必填，枚举：draft / active）
- tags（多选，最多5个标签）
- dueDate（选填，日期选择器，不能早于今天）
- coverImage（选填，图片上传，最大 2MB）

技术实现：
1. 使用 react-hook-form + zodResolver 进行表单管理
2. Zod Schema 与后端 /lib/validations/project.ts 共用（从 shared 导入）
3. 表单组件放在 /components/features/ProjectForm.tsx（新建和编辑复用）

提交流程：
1. 前端验证通过 → 按钮变为 loading 状态（禁用 + 显示 spinner）
2. 调用 POST /api/projects
3. 成功：
   - 显示 toast "项目创建成功 🎉"
   - 关闭 Modal / 跳转到项目详情页
   - 自动刷新项目列表（invalidate query cache）
4. 失败：
   - 服务端字段错误 → 显示在对应字段下方（红色提示文字）
   - 网络错误 → 显示全局 toast 错误提示
   - 保留用户已填写的内容（不重置表单）

额外要求：
- 表单有未保存更改时，关闭 Modal 弹出确认对话框
- 支持 Cmd/Ctrl + Enter 快捷键提交
- 图片上传使用拖拽 + 点击两种方式，显示上传进度
