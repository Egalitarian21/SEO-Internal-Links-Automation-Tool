# Phase 5-2：权限系统（RBAC）

## 任务描述
实现基于角色的访问控制系统（Role-Based Access Control）。
定义 admin / editor / viewer 三种角色，在 API 层和前端 UI 层
分别进行权限控制，确保用户只能执行其角色允许的操作。

---

## Prompt

实现完整的 RBAC 权限控制系统：

角色定义：
- admin：全部权限（增删改查 + 用户管理 + 系统设置）
- editor：内容权限（增改查，不能删除，不能管理用户）
- viewer：只读权限（仅查看，不能创建或修改）

第一步：数据模型更新
在 Prisma Schema 的 User 模型中添加：
- role 字段（枚举：ADMIN / EDITOR / VIEWER，默认 VIEWER）
- 生成对应的 migration

第二步：权限检查工具（/lib/permissions.ts）
创建权限矩阵和检查函数：
- PERMISSIONS 对象：定义每个角色可执行的操作
- can(user, action, resource?) 函数：检查用户是否有权限
- requirePermission(action) 中间件：API 路由权限守卫

权限矩阵示例：
- projects.create：admin, editor
- projects.delete：admin only
- projects.read：admin, editor, viewer
- users.manage：admin only

第三步：API 层权限控制
在所有 API 路由中添加权限检查：
- DELETE /api/projects/[id]：requirePermission('projects.delete')
- POST /api/projects：requirePermission('projects.create')

第四步：前端 UI 权限控制
创建 /hooks/usePermissions.ts：
- usePermissions() Hook 返回当前用户的权限列表
- <PermissionGate permission="projects.delete"> 组件：无权限时隐藏子元素
- 在 Dashboard 中，根据角色显示/隐藏操作按钮（删除、编辑等）
