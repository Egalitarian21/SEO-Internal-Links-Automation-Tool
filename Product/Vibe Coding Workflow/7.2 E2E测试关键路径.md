# Phase 6-2：E2E 测试关键路径

## 任务描述
使用 Playwright 对最核心的用户操作路径进行端到端测试。
E2E 测试不需要覆盖所有场景，只需覆盖"如果这个坏了，产品就不能用"的关键路径。
测试应该在 CI/CD 流水线中自动运行。

---

## Prompt

用 Playwright 生成核心用户路径的 E2E 测试：

测试文件位置：/e2e/

测试套件 1：认证流程（/e2e/auth.spec.ts）
- 用户注册 → 验证邮箱 → 进入 Dashboard
- 用户登录 → 跳转 Dashboard
- 未登录访问 /dashboard → 重定向到 /login
- 登出 → 跳转首页 → 无法访问 /dashboard

测试套件 2：项目核心流程（/e2e/projects.spec.ts）
完整流程测试：
1. 登录（使用测试账号）
2. 进入项目列表页
3. 点击"新建项目"按钮
4. 填写项目表单（title, description, tags）
5. 提交 → 验证成功 toast
6. 在列表中找到新建的项目
7. 点击进入详情页 → 验证数据正确
8. 点击编辑 → 修改 title → 保存
9. 验证列表中 title 已更新
10. 删除项目 → 确认弹窗 → 验证从列表消失

测试套件 3：权限验证（/e2e/permissions.spec.ts）
- viewer 账号登录 → 验证"新建"按钮不可见
- viewer 账号 → 直接访问 /projects/new → 重定向或显示无权限

配置要求：
- playwright.config.ts：配置 baseURL, 截图保存路径
- 测试前自动运行种子数据脚本
- 失败时自动截图 + 录制视频
- 生成 HTML 测试报告
