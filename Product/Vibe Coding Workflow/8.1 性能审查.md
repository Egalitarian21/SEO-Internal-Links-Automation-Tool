# Phase 7-1：性能审查

## 任务描述
在部署前对代码进行系统性的性能审查，识别并修复常见的性能问题。
重点关注数据库查询效率（N+1 问题）、前端渲染性能和资源加载优化。
目标：Core Web Vitals 全绿，LCP < 2.5s，FID < 100ms，CLS < 0.1。

---

## Prompt

对当前项目进行全面的性能审查，找出问题并提供修复方案：

数据库层面：
1. 检查所有 Prisma 查询，找出 N+1 查询问题
   - 列出所有需要添加 include 或 select 优化的查询
   - 给出优化前后的代码对比
2. 分析数据库索引使用情况
   - 检查哪些查询字段缺少索引
   - 生成 ALTER TABLE 添加索引的 SQL（或 Prisma migration）
3. 检查是否有可以用数据库聚合替代的应用层计算

前端层面：
1. 检查所有页面的 bundle size
   - 找出可以懒加载的组件（使用 dynamic import）
   - 找出可以移除的未使用依赖
2. 图片优化
   - 找出所有未使用 next/image 的 <img> 标签
   - 给出替换方案
3. 检查 React 渲染性能
   - 找出缺少 useMemo / useCallback 的高频计算
   - 找出可能导致不必要重渲染的 Context 使用

API 层面：
1. 检查是否有可以添加缓存的接口（Redis / HTTP Cache Headers）
2. 检查是否有可以合并的多个 API 请求
3. 检查大数据量接口是否有分页保护

请按「严重程度」排序输出问题清单，并逐一提供修复代码。
