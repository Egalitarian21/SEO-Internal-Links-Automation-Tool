# Phase 7-2：部署配置与 CI/CD

## 任务描述
生成完整的生产环境部署配置，包括 Vercel 部署设置和 GitHub Actions CI/CD 流水线。
CI/CD 流水线需要覆盖代码检查、测试、构建和部署四个阶段，
确保每次代码合并都经过自动化质量检查。

---

## Prompt

生成完整的部署配置和 CI/CD 流水线：

第一部分：Vercel 配置（vercel.json）
- 函数超时设置（API 路由 30s，其他 10s）
- 部署区域：iad1（美东）+ sin1（新加坡）
- 环境变量分组（production / preview / development）
- 重写规则（/api/* → serverless functions）
- 安全 Headers（CSP, X-Frame-Options, HSTS）

第二部分：GitHub Actions CI/CD（.github/workflows/）

ci.yml（每次 PR 触发）：
1. Checkout 代码
2. 安装 Node.js 20 + pnpm
3. 安装依赖（使用缓存）
4. 运行 ESLint + TypeScript 类型检查
5. 运行 Vitest 单元测试（生成覆盖率报告）
6. 构建项目（验证 build 不报错）
7. 上传测试覆盖率到 Codecov

deploy.yml（main 分支 merge 后触发）：
1. 运行完整 CI 流程
2. 运行 Playwright E2E 测试（针对 Preview 环境）
3. 部署到 Vercel Production
4. 运行 Smoke Test（验证关键页面可访问）
5. 发送部署通知到 Slack

第三部分：环境变量清单（docs/env-vars.md）
列出所有环境变量：
- 变量名
- 用途说明
- 在哪里获取
- 是否必填
- 示例值（脱敏）
