## 1. 内链的定义与作用

内部链接（Internal Links）是指同一站点内不同页面之间的链接，例如从一篇博客链接到同域名下的产品页、集合页或另一篇博客文章。

内链的核心作用：

1. 帮助 Google 发现更多页面，降低孤立页面数量。
2. 传递页面权重，帮助搜索引擎判断不同页面的重要性。
3. 建立页面之间的语义关系，帮助搜索引擎理解主题集群。
4. 引导用户继续阅读，提升站内浏览深度，降低跳出率。
5. 提高爬虫抓取效率。

## 2. 内链数量标准

每篇博客正文内至少应包含以下内链：

| 类型            | 目标页面                     |
| --------------- | ---------------------------- |
| 产品/集合页内链 | 产品页、产品集合页           |
| 博客/信息页内链 | 与当前主题相关的站内其他博客 |
| 发布后反向内链  | 从旧博客补加指向新博客的内链 |

注意：

- 导航栏、页脚、侧边栏中的链接不计入正文内链数量。
- 重点统计博客正文中的上下文内链（Context Internal Links）。
- 同一个产品或同一个目标信息页，在同一篇文章中尽量只链接一次。

## 3. 内链添加位置规范

1. 文章第一段不添加任何链接，避免影响跳出率。
2. H1-H6 标题中不添加锚文本链接。
3. 标题和列表冒号前的内容不添加链接。
4. 信息类链接应放在段落解释完毕之后，而不是段首。
5. 整篇文章的链接分布应尽可能均衡，不要集中堆在某一小段。
6. 链接应出现在语义自然、对读者有帮助的位置，不能为了凑数量硬加。

## 4. 目标页面选择规范

### 4.1 产品/集合页内链

适合链接到产品或集合页的情况：

- 当前段落提到具体产品、产品类型、购买意图或商业词。
- 出现大词或类目词时，优先链接到集合页。
- 出现商业词，如“购买 xx”“xx 产品”“xx 解决方案”等，可链接到产品页或集合页。
- 出现“品牌词 + 通用词”时，推荐将整体作为锚文本链接到对应集合页。

补充规则：

- 复数形式或类目泛词通常链接到产品集合页。
- Shopify 产品链接如果形如 `{域名}/collections/**/products/**`，应删除中间的 `collections/**`，只保留 `{域名}/products/**`，因为 Shopify canonical 通常指向 `/products`。

### 4.2 博客/信息页内链

适合链接到其他博客的情况：

- 当前文章中的某个概念、分类点或步骤，可以由另一篇博客展开解释。
- 某个段落提到的内容，与已有博客主题高度相关。
- 指南类文章中的某个分类点可以延伸到更详细的教程或说明文章。

不适合的情况：

- 信息性短语，如“一项研究表明”，不应链接到商业页面。
- 与当前上下文无明显关系的页面，不应强行链接。
- 不要用同一个锚文本在多个地方反复指向同一个网址。

## 5. 锚文本选择原则

锚文本是用户可以点击的链接文字，也是搜索引擎判断目标页面主题的重要信号。

优先选择：

1. 必须是博客文中出现的，可以具体查找到文本
2. 与目标页面主题高度相关的词组。
3. 当前句子中自然存在的名词、名动词组或完整短语。
4. 能准确说明目标页面内容的词，而不是模糊的“点击这里”“了解更多”。
5. 语义相关词、同义词、长尾词，帮助锚文本保持多样化。

避免选择：

1. 单个单词作为锚文本。
2. 被截断或拆分的单词。
3. 生硬堆砌多个关键词的短语。
4. 与目标页面无关的词。
5. 过长、难以自然嵌入正文的页面标题。

特殊情况：

- 如遇“全称 (缩写)”格式，应将整体作为锚文本。
- 图片链接需要设置准确的 `alt` 属性，`alt` 会承担类似锚文本的作用。

## 6. 常见锚文本类型与使用建议

| 锚文本类型 | 示例                        | 使用建议                     |
| ---------- | --------------------------- | ---------------------------- |
| 完全匹配型 | 上海公司注册                | SEO 信号强，但不要过度使用   |
| 部分匹配型 | 上海注册公司的步骤          | 推荐使用，兼顾相关性和自然度 |
| 品牌词型   | 华为官网                    | 安全，但需与上下文相关       |
| 通用型     | 点击这里、了解更多          | SEO 价值弱，应少用           |
| 语义相关型 | 瘦身餐、健康饮食计划        | 推荐使用，可提升多样性       |
| 页面标题型 | 2025 年最新市场营销趋势分析 | 清晰但可能过长，需自然嵌入   |
| URL 型     | `https://www.example.com`   | 不便阅读，尽量少用           |
| 图片链接   | 图片 + `alt`                | 必须设置准确 `alt`           |

锚文本组合建议：

- 不要只使用完全匹配型锚文本。
- 应混合使用部分匹配、语义相关、页面标题、品牌词等类型。
- 同一目标页面在不同文章中可使用不同但相关的锚文本。
- 锚文本应简洁明了，通常包含 1-2 个核心词即可。

## 7. 锚文本与链接相关性检查

添加内链前，需要确认三层相关性：

1. 锚文本本身与目标页面主题相关。
2. 锚文本所在段落与目标页面主题相关。
3. 目标页面能进一步解释、补充或承接当前内容。

如果找不到自然锚文本，可以在原段落中补充一两句自然过渡句，引出要链接的文章，但不能为了凑内链破坏文章流畅度。

## 8. 寻找站内博客的方法

可通过以下方式寻找适合添加的博客内链：

1. 查看网站 `sitemap.xml`，筛选带有 `/blog`、`/blogs`、`/news` 等路径的 URL。
2. 使用站点地图解析工具查看站内博客列表。
3. 查看内容管理表或团队内部文档，优先选择团队自己产出的相关博客。
4. 在 GSC 中查看网页，按博客集合页名称筛选。
5. 使用 Google site 搜索：`site:域名 相关词`。

选择博客时优先考虑：

- 与当前文章主题高度相关。
- 能详细解释当前锚文本中的概念。
- 不是重复主题或会造成内部竞争的页面。
- 是希望继续提升权重的重点页面。

## 9. 添加内链操作流程

### 9.1 添加产品/集合页链接

1. 从官网中找到与文章内容匹配的产品页或集合页。
2. 复制目标 URL。
3. 在文章中找到准确、自然的产品词、类目词或商业词作为锚文本。
4. 选中锚文本并添加链接。
5. 检查同一产品在同一篇文章中是否重复链接，尽量只保留一次。

### 9.2 添加博客链接

1. 找到与当前文章或某一段内容相匹配的站内博客。
2. 复制目标博客 URL。
3. 在当前文章中选中准确、相关、自然的锚文本。
4. 添加链接。
5. 检查同一个目标博客在本文中是否只添加一次。

## 10. 发布前检查清单

发布前逐项检查：

- 产品/集合页内链是否不少于 3 条。
- 相关博客内链是否不少于 3 条。
- 首段是否没有添加任何链接。
- H 标签中是否没有添加链接。
- 锚文本是否自然、相关、未被截断。
- 是否避免使用单个单词、通用词或关键词堆砌作为锚文本。
- 同一目标信息页是否只链接一次。
- 链接是否分布均衡。
- 超链接样式是否清晰可辨，如颜色、下划线等。
- Shopify 产品 URL 是否使用 canonical 对应的 `/products/` 路径。

## 11. 发布后内链维护

内链不是发布一次就完成的工作，需要持续维护。

1. 新文章发布后，应在 3 篇以上旧文章中补加指向新文章的内链。
2. 建议每季度使用 Screaming Frog 或 Ahrefs Site Audit 检查站内 404 内链。
3. 如果文章合并、URL 修改或页面删除，必须同步更新所有指向旧 URL 的内链，并做好 301 重定向。
4. 建议建立“内链台账”，记录每篇文章的核心出站内链和入站内链，方便批量更新和断链排查。

## 12. 常见错误

1. 只用一种锚文本类型，尤其是反复使用完全匹配型锚文本。
2. 在锚文本中堆砌关键词。
3. 链接到与上下文无关的页面。
4. 多次使用完全相同的锚文本指向同一个 URL。
5. 为了凑数量在首段、标题或不自然的位置加链接。
6. 只添加产品链接，忽略相关博客链接。
7. 发布新文章后没有从旧文章补反向内链。
8. 页面更新或合并后没有同步修改旧内链。
